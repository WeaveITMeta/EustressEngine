# Manufacturing Deal Structure — Eustress Manufacturing Program

Specification for the equity distribution system and manufacturing deal pipeline built into
the Eustress Studio Workshop Panel (System 0: Ideation).

---

## Overview

When a product completes the ideation pipeline (steps 0–8: brief → patent → SOTA →
requirements → mesh → parts → sim scripts → UI → catalog), two additional steps
are automatically proposed:

| Step | Artifact | Purpose |
|---|---|---|
| **Step 9** | `DEAL_STRUCTURE.md` | Equity split, royalty terms, unit economics, IP governance |
| **Step 10** | `LOGISTICS_PLAN.md` | Pilot program, warehousing, fulfillment, regulatory |

Both are generated by Claude using the `ideation_brief.toml` as context. Each requires
explicit user approval (MCP command card) before spending BYOK credits (~$0.04 each).

---

## The Manufacturing Program Model

### Core Principle

The Eustress Manufacturing Program is a **self-funding manufacturing fund**. Every product
that ships through the program contributes a royalty percentage back into the fund. That
fund pays for:

- Future pilot programs for new inventors
- Warehousing capacity expansion
- Logistics partner negotiations
- Manufacturing tooling and tooling amortization
- The feedback loop: real-world usage data → System 3 fitness improvement

This creates a **compounding flywheel**: successful products fund the next cohort of
products, improving the entire Eustress product ecosystem over time.

```
│  Logistics Partner                 10%  ██░░░░░░░░░░░░░░░  │
│  Reserve Pool                       5%  █░░░░░░░░░░░░░░░░  │
│                                   ───                       │
│  Total                            100%                      │
└─────────────────────────────────────────────────────────────┘
```

### Stakeholder Roles

| Stakeholder | Role | Default % | Vesting |
|---|---|---|---|
| **Inventor** | IP owner, product creator, technical authority | 40% | Immediate |
| **Eustress Manufacturing Program** | Manufacturing fund, infrastructure, distribution network, platform | 25% | Immediate |
| **Bliss Contributor Pool** | Ecosystem contributors (builders, scripters, educators) who enabled the product | 20% | Distributed monthly to active contributors |
| **Logistics Partner** | 3PL operations, warehousing, fulfillment, last-mile | 10% | 12-month cliff, 24-month vest |
| **Reserve Pool** | Future co-investors, advisors, strategic distribution partners | 5% | Board discretion |

### Adjustment Factors

Claude adjusts the default split based on signals in the `ideation_brief.toml`:

| Signal | Adjustment |
|---|---|
| High BOM cost (>$50/unit) | Eustress Program % increases (more capital at risk) |
| Strong IP (>20 patent claims) | Inventor % increases |
| Simple commodity product | Logistics Partner % may increase |
| Exotic/novel manufacturing required | Eustress Program % increases |
| Product already has sales traction | Inventor % increases |

### Validation

`DealStructure::validate_equity_sum()` enforces that all `EquityStake.percentage` values
sum to exactly 100.0% (within 0.01% float tolerance). A failed validation surfaces as an
error message in the Workshop Panel conversation before the artifact is saved.

## Royalty Structure

Royalties are percentages of **net sales** (gross revenue minus returns, chargebacks,
and sales tax), paid out monthly:

| Royalty | Recipient | Default % | Purpose |
|---|---|---|---|
| **Manufacturing Program Royalty** | Eustress Manufacturing Program fund | 8% | Funds future pilot programs and infrastructure |
| **Bliss Contributor Royalty** | Bliss distribution pool | 5% | Monthly payouts to all ecosystem contributors |
| **Inventor Royalty** | Inventor | 5% | Ongoing compensation above equity distributions |

These are **in addition to** equity distributions, not instead of. An inventor with 40%
equity receives equity dividends proportional to profit, plus 5% of every unit sold as
a royalty stream. Bliss contributors collectively receive 20% equity distributions plus
5% royalty from all products.

### Per-Unit Royalty Calculation

```
unit_price_usd = $49.00
manufacturing_program_royalty_pct = 8%
bliss_contributor_royalty_pct = 5%
inventor_royalty_pct = 5%

manufacturing_royalty_per_unit = $49.00 × 0.08 = $3.92
bliss_royalty_per_unit         = $49.00 × 0.05 = $2.45
inventor_royalty_per_unit      = $49.00 × 0.05 = $2.45
total_royalty_outflow          = $3.92 + $2.45 + $2.45 = $8.82 per unit (18% total)
```

### Manufacturing Program Fund Contribution

After a full pilot batch ships, the fund contribution is:

```
pilot_fund_contribution = manufacturing_royalty_per_unit × pilot_minimum_units
                        = $3.92 × 1,000 = $3,920
```

This $3,920 re-enters the fund and becomes available for the next inventor's pilot.

### Bliss Contributor Pool Contribution

After a full pilot batch ships, the Bliss pool receives:

```
bliss_pool_contribution = bliss_royalty_per_unit × pilot_minimum_units
                        = $2.45 × 1,000 = $2,450
```

This $2,450 is distributed monthly to all Bliss contributors based on their contribution scores (building, scripting, teaching, collaboration). Contributors who helped build the ecosystem that enabled this product now earn from its success.

## Two-Layer Distribution Model

The Eustress Manufacturing Program uses a **two-layer distribution system** to balance immediate cash flow with long-term wealth creation:

### Layer 1: Royalties (18% of Gross Revenue)

Paid **immediately** on each sale, before expenses:

| Stakeholder | Royalty % | Purpose |
|-------------|-----------|---------|
| Manufacturing Program | 8% | Funds future pilot programs and infrastructure |
| Bliss Contributor Pool | 5% | Monthly payouts to ecosystem contributors |
| Inventor | 5% | Recurring income stream |
| **Total** | **18%** | Deducted from gross revenue |

**Key characteristics:**
- Paid on **gross revenue** (before expenses)
- Guaranteed cash flow regardless of profitability
- Protects downside for all stakeholders
- Ensures Manufacturing Program and Bliss Pool remain funded

### Layer 2: Net Profit Distribution (Equity-Based)

Paid **quarterly or annually** after all expenses settled:

| Stakeholder | Equity % | Share of Net Profit |
|-------------|----------|---------------------|
| Inventor | 40% | 40% of net profit |
| Manufacturing Program | 25% | 25% of net profit |
| Bliss Contributor Pool | 20% | 20% of net profit |
| Logistics Partner | 10% | 10% of net profit |
| Reserve Pool | 5% | 5% of net profit |
| **Total** | **100%** | All net profit distributed |

**Key characteristics:**
- Paid on **net profit** (revenue - royalties - expenses)
- Depends on profit margin (could be $0 if product loses money)
- Rewards efficient operations and high-margin products
- Aligns incentives: everyone wants to minimize expenses

## Complete Distribution Example

### Scenario A: High-Margin Product (42% net margin)

**Product:** V-Cell 4680 Battery  
**Sales:** 10,000 units at $50/unit

#### Revenue Flow

```
Gross Revenue:                              $500,000

Layer 1: Royalties (deducted first)
  Manufacturing Program (8%):                -$40,000
  Bliss Pool (5%):                           -$25,000
  Inventor (5%):                             -$25,000
  Total Royalties (18%):                     -$90,000
                                            ─────────
Revenue After Royalties:                    $410,000

Expenses:
  COGS (10,000 × $12.50):                  -$125,000
  Warehousing:                                -$5,000
  Fulfillment (pick/pack/ship):              -$20,000
  Marketing:                                 -$15,000
  Returns/refunds (5%):                      -$25,000
  Overhead (admin, legal, insurance):        -$10,000
  Total Expenses:                           -$200,000
                                            ─────────
Net Profit:                                 $210,000 (42% margin)
```

#### Layer 2: Profit Distribution

```
Inventor (40%):              $210,000 × 0.40 = $84,000
Manufacturing Program (25%): $210,000 × 0.25 = $52,500
Bliss Pool (20%):            $210,000 × 0.20 = $42,000
Logistics Partner (10%):     $210,000 × 0.10 = $21,000
Reserve Pool (5%):           $210,000 × 0.05 = $10,500
```

#### Total Payout Per Stakeholder

| Stakeholder | Royalty (Layer 1) | Profit (Layer 2) | Total | % of Revenue |
|-------------|-------------------|------------------|-------|--------------|
| **Inventor** | $25,000 | $84,000 | **$109,000** | 21.8% |
| **Manufacturing Program** | $40,000 | $52,500 | **$92,500** | 18.5% |
| **Bliss Pool** | $25,000 | $42,000 | **$67,000** | 13.4% |
| **Logistics Partner** | $0 | $21,000 | **$21,000** | 4.2% |
| **Reserve Pool** | $0 | $10,500 | **$10,500** | 2.1% |
| **TOTAL** | $90,000 | $210,000 | **$300,000** | 60.0% |

**Remaining $200,000 (40%) went to expenses.**

## Why This Model Works

### 1. Royalties Protect Downside
- Even break-even products fund Manufacturing Program and Bliss Pool
- Inventor gets minimum 5% income stream
- Ensures ecosystem sustainability

### 2. Profit Distribution Rewards Upside
- High-margin products generate significant equity payouts
- Aligns incentives: everyone wants to maximize profit
- Inventor can earn 21.8% of revenue on successful products

### 3. Risk is Shared Fairly
- Logistics Partner only earns if product is profitable (incentivizes good fulfillment)
- Reserve Pool only earns if product succeeds (incentivizes strategic support)
- Manufacturing Program and Bliss Pool protected by royalties

### 4. Simple and Predictable
- Fixed percentages (no complex formulas)
- Easy to calculate and audit
- Transparent for all stakeholders

## Unit Economics

The deal document calculates and displays a full unit economics table:

| Metric | Formula |
|---|---|
| Suggested Retail Price | From `DealStructure.unit_price_usd` |
| Estimated Unit Cost | BOM total + 35% assembly + 15% logistics + 10% returns reserve |
| Gross Margin per Unit | `unit_price_usd - unit_cost_usd` |
| Gross Margin % | `gross_margin_usd / unit_price_usd × 100` |
| Mfg Program royalty | `unit_price_usd × manufacturing_program_royalty_pct / 100` |
| Inventor royalty | `unit_price_usd × inventor_royalty_pct / 100` |
| Net to equity pool | Gross margin minus royalties minus overhead |

## Pilot Program

Before full production is unlocked, every product must complete a pilot:

### Minimum Requirements

| Parameter | Default | Notes |
|---|---|---|
| Batch size | 1,000 units | Minimum 500; electronics recommend 1,000 |
| Duration | 12 weeks | Time to collect meaningful telemetry and returns data |
| Go/no-go criteria | Defined per product | Return rate, rating, repurchase intent |

### Success Criteria Examples

- Return rate < 5%
- Average customer rating > 4.2 stars
- Repurchase intent > 80% (survey)
- Zero safety incidents
- IoT telemetry uptime > 99% (for connected products)

### Kill Conditions

The pilot is stopped and the product is returned to ideation if:
- Return rate > 15% in the first 4 weeks
- Any safety-related returns regardless of rate
- Manufacturing defect rate > 3%
- Customer NPS below 20

## Logistics Pipeline

Three phases from pilot to scale:

```
Phase 1: Pilot (12 weeks, 500–1,000 units)
  └── Validate product-market fit
  └── Collect telemetry and return data
  └── Qualify fulfillment partner
  └── Measure go/no-go criteria
        │
        ▼  (criteria met)
Phase 2: Limited Production (6 months, 5,000–10,000 units)
  └── Optimize pick/pack/ship cost
  └── Negotiate volume freight rates
  └── Expand to additional SKUs or variants
  └── Build safety stock buffer
        │
        ▼  (demand signals confirm)
Phase 3: Full Production (ongoing, 10,000+/month)
  └── Scale with demand
  └── Add geographic warehouse nodes
  └── Automate reorder via OMS integration
  └── Feed real-world data back to System 3 (fitness improvement)
```

### Warehousing Models

| Model | Best for | Pros | Cons |
|---|---|---|---|
| **3PL (ShipBob, Flexport)** | Most products | No fixed cost, scales up/down | Per-unit fees |
| **Amazon FBA** | Consumer retail focus | Prime badge, discovery | Lose direct customer relationship |
| **Own warehouse** | High-volume, controlled products | Full control, lower unit cost at scale | High fixed cost |
| **Consignment** | Specialty retail | No upfront cost | Revenue delayed, complexity |

Default recommendation: **3PL** for pilot, reassess at Phase 2.

### Inventory Parameters

```
safety_stock  = avg_daily_demand × lead_time_days × 1.5
reorder_point = avg_daily_demand × lead_time_days + safety_stock
EOQ           = sqrt(2 × annual_demand × order_cost / holding_cost_per_unit)
```

### Fulfillment Cost Breakdown

| Component | Typical Cost |
|---|---|
| Pick + pack | $2.50/order |
| Domestic shipping (standard) | $5–8/order |
| Inbound freight (from mfr) | ~8% of unit cost |
| Warehousing (allocated) | ~$0.50/unit/month |
| Returns processing | ~$3.00/return |

## Intellectual Property Terms

- IP remains owned by the inventor
- Eustress Manufacturing Program receives an **exclusive manufacturing license** for the
  pilot period (12 weeks)
- License converts to **non-exclusive** after pilot if production targets are met
- Patent filing costs split 50/50 between inventor and Manufacturing Program
- Manufacturing Program may not sublicense without inventor written consent

---

## Governance

| Decision Type | Authority |
|---|---|
| Product design changes | Inventor (final say) |
| Manufacturing process | Eustress Manufacturing Program (final say) |
| Retail pricing | Joint approval required |
| Distribution channel additions | Joint approval required |
| Hiring logistics partners | Eustress Manufacturing Program with inventor notification |
| Dispute resolution | Binding arbitration |

---

## Exit Terms

| Scenario | Terms |
|---|---|
| **Inventor buyout** | Inventor may buy out Manufacturing Program stake at 3× invested capital after 24 months |
| **Manufacturing Program exit** | May sell stake to pre-approved third parties; inventor has right-of-first-refusal at same price |
| **Product discontinuation** | Joint decision; IP reverts fully to inventor; remaining inventory liquidated with proceeds split by equity % |
| **Acquisition** | Proceeds distributed by equity %; inventor receives inventor royalty buyout at 5× annual royalty run-rate |

## Rust Data Model

The deal is encoded in `DealStructure` (in `mod.rs`) which is embedded in `IdeationBrief`:

```rust
pub struct DealStructure {
    pub title: String,
    pub equity_splits: Vec<EquityStake>,
    pub manufacturing_program_royalty_pct: f64,   // default 8.0
    pub inventor_royalty_pct: f64,                 // default 5.0
    pub unit_price_usd: f64,
    pub unit_cost_usd: f64,
    pub pilot_minimum_units: u32,                  // default 1000
    pub pilot_geography: String,
    pub term_validity_months: u32,                 // default 6
    pub notes: String,
    pub logistics: Option<LogisticsPlan>,
}

pub struct EquityStake {
    pub stakeholder: String,
    pub role: String,
    pub percentage: f64,                           // must sum to 100.0
    pub vesting_cliff_months: Option<u32>,
    pub vesting_period_months: Option<u32>,
}
```

Validation: `DealStructure::validate_equity_sum()` returns `Err` if stakes do not sum to
100.0% ± 0.01.

The `LogisticsPlan` struct nested inside covers pilot, warehousing, fulfillment, and
regulatory fields — all serializable to TOML for persistence and git diffing.

## Pipeline Integration

Steps 9 and 10 appear at the end of the Workshop Panel step list:

```
Step 0   Normalize brief              ✓
Step 1   Patent draft                 ✓
Step 2   SOTA validation              ✓
Step 3   Requirements                 ✓
Step 4   Mesh generation              ✓
Step 5   Part files                   ✓
Step 6   Rune sim scripts             ✓
Step 7   UI + UI scripts              ✓
Step 8   Catalog entry                ✓
Step 9   Deal structure               ← NEW  ~$0.04
Step 10  Logistics plan               ← NEW  ~$0.04
```

Total pipeline cost: ~$0.37 BYOK per product (all 11 steps).

Both steps produce Markdown files in `Space/Workspace/{product}/`:
- `DEAL_STRUCTURE.md`
- `LOGISTICS_PLAN.md`

These are shown as artifact cards in the Workshop Panel conversation and open directly
in the Studio center tab viewer when clicked.
